# Helios v2 Plan (Baseline-First)

## Goal
Finish remaining v1 runtime wiring gaps first, then complete and modernize the `@helios/nestjs` split (Phase 9) from the current scaffold.

## Ground Truth (repo state)
- Workspace is already configured at root (`package.json`) for `ts-port`, `ts-port-nestjs`, `ts-port-app`.
- `ts-port-nestjs/` exists but is skeletal (`src/HeliosInstanceDefinition.ts`, `src/TransactionExceptions.ts`, empty test tree).
- Core package rename is in progress (`ts-port/package.json` uses `@helios/core`), but core still contains NestJS code/exports.
- Discovery pieces exist (`ts-port/src/discovery/HeliosDiscovery.ts`, `ts-port/src/internal/cluster/ClusterJoinManager.ts`) but startup wiring is incomplete: `ts-port/src/instance/impl/HeliosInstanceImpl.ts` still bootstraps peers from TCP-IP members directly.
- File-based config parsing is incomplete for network/join/discovery: `ts-port/src/config/ConfigLoader.ts` currently parses only instance name + maps.

## Execution Plan

### 1) Reconcile plan/documentation before coding
- Update `plans/TYPESCRIPT_PORT_PLAN.md` so block status lines match actual implementation state (remove contradictory TODO vs DONE rows).
- Keep Phase 9 as active workstream; explicitly mark v1 baseline items that still need runtime hardening.
- Acceptance: no contradictory checkboxes/commit lines for the same block.

### 2) Finish v1 discovery bootstrap path (first priority)
- Extend config ingestion in `ts-port/src/config/ConfigLoader.ts` to parse:
  - `network.port`
  - `network.join.tcpIp`
  - discovery provider config (`aws/gcp/azure/k8s/eureka/discovery` shape supported by `JoinConfig`).
- Use `JoinConfig.verify()` as the single join-exclusivity gate; fail fast on invalid multi-join inputs.
- Acceptance: file-based config can drive discovery-capable join settings without manual object construction.

### 3) Wire discovery into runtime startup
- Integrate `ClusterJoinManager` into `ts-port/src/instance/impl/HeliosInstanceImpl.ts` startup path so peer resolution is `JoinConfig`-driven (not TCP-IP-only).
- Keep deterministic fallback order: explicit TCP-IP members first, then typed/aliased discovery providers.
- Preserve current transport behavior (`TcpClusterTransport`) and connect lifecycle.
- Acceptance: startup resolves peers via join config and attempts connects from resolved member list.

### 4) Add/expand v1 integration coverage for discovery path
- Extend `ts-port/test/internal/cluster/ClusterJoinManagerTest.ts` for typed discovery + alias coverage.
- Extend `ts-port/test/factory/HeliosFactoryTest.ts` with file-based network/join/discovery cases.
- Add instance-level bootstrap integration test (new):
  - `ts-port/test/instance/HeliosInstanceDiscoveryBootstrapTest.test.ts`
  - validates config -> join resolution -> startup connect attempts.
- Acceptance: discovery path is validated at config, join manager, and instance bootstrap levels.

### 5) Complete Phase 9.0 package extraction from current scaffold
- Populate `ts-port-nestjs/src/` by moving NestJS integration files from `ts-port/src/nestjs/`.
- Move NestJS tests from `ts-port/test/nestjs/` into `ts-port-nestjs/test/`.
- Add missing package wiring files:
  - `ts-port-nestjs/src/index.ts`
  - `ts-port-nestjs/tsconfig.build.json`
- Normalize imports:
  - intra-package NestJS imports to relative paths
  - core imports to `@helios/core/*`.
- Acceptance: `ts-port-nestjs` builds/tests independently; no missing module references.

### 6) Enforce clean core/nestjs separation
- Remove NestJS runtime deps from `ts-port/package.json`.
- Remove NestJS re-exports from `ts-port/src/index.ts`.
- Delete old `ts-port/src/nestjs/` and `ts-port/test/nestjs/` only after parity is green in `ts-port-nestjs`.
- Update any consumers relying on old paths (notably `ts-port-app` aliases/imports where needed).
- Acceptance: `@helios/core` has zero NestJS coupling; `@helios/nestjs` is the sole NestJS surface.

### 7) Implement Phase 9 modernization blocks 9.1-9.4
- `ConfigurableModuleBuilder` migration for `HeliosModule`.
- Add convenience decorators (`@InjectHelios`, `@InjectMap`, `@InjectQueue`, etc.) and token helpers.
- Add `registerAsync` support for cache/transaction modules.
- Remove static singleton transaction manager resolution and switch `@Transactional` to DI/module-scoped resolution.
- Primary files:
  - `ts-port-nestjs/src/HeliosModule.ts`
  - `ts-port-nestjs/src/HeliosCacheModule.ts`
  - `ts-port-nestjs/src/HeliosTransactionModule.ts`
  - `ts-port-nestjs/src/HeliosTransactionManager.ts`
  - `ts-port-nestjs/src/Transactional.ts`
  - new `ts-port-nestjs/src/decorators/*` and module-definition files.
- Acceptance: modern async registration and DI-based transaction behavior are fully tested.

### 8) Implement Phase 9 feature blocks 9.5-9.9
- Add health module/integrator (`@nestjs/terminus` optional peer).
- Add method decorators (`@Cacheable`, `@CacheEvict`, `@CachePut`).
- Add event bridge (`@nestjs/event-emitter` optional peer).
- Migrate to symbol-based tokens and lifecycle shutdown hooks.
- Finalize exports/docs and dry-run publish for both packages.
- Acceptance: `@helios/nestjs` is publishable and feature-complete per Phase 9 checklist.
